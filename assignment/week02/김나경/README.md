### 분석 내용

- `통계 → HTTP → Packet Counter → HTTP Response Packets`
- 주요 상태 코드(200, 301, 404 등) 확인 및 의미 간단히 정리

## 목표

- Wireshark를 이용하여 PDF 파일 다운로드 패킷 캡처
- HTTP 상태 코드 분석

### 1. Wireshark 실행 및 캡처 시작

wifi를 사용 중이므로 해당 인터페이스를 선택하고 캡처를 시작하였다.

### 2. 디스플레이 필터 설정

Wireshark 상단 필터창에 `http`를 입력하여 HTTP 통신 패킷만 화면에 보이도록 필터링하였다.

### 3. 사이트 접속 및 암호화 해제

주소창에 `https` 대신 `http://www.yongin.go.kr`을 직접 입력하여 용인시청 홈페이지에 접속하였다.

### 4. PDF 파일 다운로드

용인시청 홈페이지 내에서 임의의 PDF 파일을 다운로드하며 통신 과정을 캡처하였다.

### 5. 상태 코드 분석

통계(Statistics) -> HTTP -> 패킷 카운터(Packet Counter) 메뉴를 통해 확인한 응답 패킷(HTTP Response Packets) 결과는 다음과 같다.

- 2xx: 정상 응답 (200 OK)
- 3xx: 리다이렉션 발생 (302)
- 4xx, 5xx: 오류 없음

  총 67개의 응답 패킷 중, 통신 성공을 의미하는 200 OK가 66개로 가장 많았고 정상적으로 파일이 다운로드됨을 확인했습니다. 또한 접속 과정에서 임시 이동을 뜻하는 302 Found (리다이렉션)가 1건 발생했습니다. 404 같은 클라이언트 오류는 발생하지 않았습니다!
